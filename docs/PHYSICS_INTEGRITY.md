# PHYSICS_INTEGRITY.md — 「これは物理だ」と言える誠実さ

**この原則は、すべての機能・成果数・見栄えより優先する。**

---

## 1. 必ず区別するもの

- 実際の物理シミュレーション ／ 説明用の軽量デモ ／ 可視化 ／ 解釈 ／ 比喩 ／ 仮説 ／ 既知理論の再現 ／ 新しい創発候補。

同じ「成功」に押し込めない。特に**既知理論の再現**と**入れていない現象の創発**を分ける（§役割）。

---

## 2. 物理として主張するための最低条件

- 方程式が明示されている。
- 初期条件・境界条件が保存されている。
- 数値法・時間刻み dt・格子幅 dx が保存されている。
- 保存量または収支が測定されている。
- 解像度依存性が確認されている。
- 複数 seed で再現性を確認している。
- 結果を出すための後処理改変がない。
- 可視化と実データの対応が明示されている。
- **出なかったものも記録されている。**
- 3D 正式結果と 2D 候補が混同されていない。

## 3. 禁止事項

- 結果を良く見せるための無断クリッピング。
- 保存量を壊す補正。
- **結果を見た後で成功条件を変更する。**
- 成功 run だけを残す。
- 実際には測っていない量を主張する。
- 解釈を測定結果として表示する。
- 2D 結果を 3D 結果として表示する。
- 軽量デモを本番シミュレーションとして見せる。
- 見栄えのために存在しない流れ・粒子を追加する。

---

## 4. 役割（E / V / S / N / F / Q）

GREEN 一色をやめ、**役割**を付す。役割は優劣でなく種類。

| role | 意味 |
|---|---|
| **E** — Emergence | 未分化/単純な状態から、対象構造が**入れずに**自然に生じることを調べた。7＋8 監査を全通過。 |
| **V** — Validation | 既知理論・解析解・実験値を正しく再現した（硬い検証。**E に劣らない**）。 |
| **S** — Synthesis | 設計した結合・閉環が、どう振る舞うかを調べた（足場付き。oracle/手配線/ラベルあり）。 |
| **N** — Negative | 求めた現象が出ない、または原理的障害があることを示した。 |
| **F** — Frontier | 重要だが、まだ測定・収束・解釈が固まっていない観測。 |
| **Q** — Quarantine | 主張とコードが不一致、または結論が埋め込まれている疑い。**修正まで正式判定を保留。** |

一つの実験は複数役割を持ってよい（primary / secondary）。

---

## 5. 確信度（Type A–D）と claim tier

**確信度（Type A–D）は役割と別軸**として残す（既存の台帳）。

**claim tier**（主張の強さ）：
- `measured` — 直接測定した。
- `observed` — 観測したが解釈の余地。
- `established` — 既知理論/解析解として確立された結果を忠実に再現した（V の典型）。
- `interpretive` — 測定からの解釈。
- `analogy` — 比喩（生命語はここ）。
- `frontier` — 未解決。

各実験は role・confidence・claim_tier を独立に持つ：
```yaml
role: {primary: E, secondary: [V]}
confidence: A
claim_tier: [measured]
audit: {physics_integrity: passed, no_target_encoding: passed}
```

## 6. 第8監査（最重要の構造修正）

**追加ルール：評価ゲートと初期条件は、結論と同型の因果構造を符号化してはならない。**

失敗の多くは「if 文で"死ね/協力しろ"と書いた」ことではなく、**結論と同型の因果を、方程式・初期条件・評価ゲートに埋め込んでいる**こと。各実験で機械的に問う：

1. **初期条件に、証明したい量そのもの（またはその関数）が入っていないか?**（例：サイズ ~ √c を証明するのに初期サイズを √c に比例させる。）
2. **ゲートが、結論の因果を直接 if 判定していないか?**（例：巻き→gain を配線し、それをゲートにする。）
3. **ゲートの閾値が、対照（null / 線形 / ランダム）でも通ってしまわないか?**（例：分化ゲートが、分化しないはずの線形設定でも通る。）
4. **「創発した」量が、実は別の入力の代数的な言い換えでないか?**（例：spacing = L/√N。）

判定：`target_encoded=true` なら role=**Q**（隔離）または **S**（足場付きと明示）。CI で静的に検査。

**必須の対照（第8監査の実践）：**
- 主張が「機構 X が現象 Y を起こす」なら、**X を切った対照で Y が消える**ことを並べる。
- 多様性の主張は**有界空間＋中立（選択なし）対照**（無界拡散を多様性と誤判定しない）。
- well-mixed vs spatial の比較は**同じ総量（mass-matched）**で。

## 7. 複数の忠実実現（robustness の資産）

同じ創発事実が**異なる本物の場の方程式から独立に出る**なら、それは現象が実在する証拠（一模型のクセでない）。`docs/TRUST_MAP.md` に記す。

例：分化＝Cahn-Hilliard 相分離 **も** morphogen/Turing **も**；協力＝双安定フロント（安定性）**も** 生態的公共財（持続）**も**。

---

## 8. 「自然に創発した」の記録

各 Room に：
```yaml
natural_emergence:
  started_from_time_zero: true
  target_shape_seeded: false
  runtime_interventions: 0
  target_dependent_rules: false
  target_dependent_stopping: false
  target_dependent_clipping: false
  level_detected_by_measurement: true
```

**外部駆動について：** 時間依存の駆動を全て禁止するわけではない（温度差・光・化学ポテンシャル差・周期駆動・物質流入は自然界にある）。ただし**始原/環境条件として事前定義され、結果を見て途中変更されていない**こと。表示で区別：`autonomous / externally_driven / time_programmed_environment / runtime_controlled`。**runtime_controlled は正式な自律創発と区別する。**

## 9. 強い語の扱い

「生命」「脳」「身体」「宇宙」「意識」等は、**正式な定義（測定量）を満たさない限り使わない**。それまでは「身体候補」「神経様関係候補」「植物様形態候補」など**限定名称**を使う。新しい創発は**既存の意味ラベルでなく測定量から判定**する。「同じ数学 ≠ 同じもの」。
