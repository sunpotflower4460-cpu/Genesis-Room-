# DIMENSION_POLICY.md — 2D 探索 / 3D 正式 / 次元移行監査

**2D と 3D では、同じ方程式でも現象が変わりうる。** 特に渦の形状/安定性・欠陥の種類・トポロジー・乱流・エネルギーカスケード・循環構造・小胞の形状変化・分裂経路・絡み/結び目/渦線・境界曲率・崩壊モードは次元依存性が大きい。**2D で成功したことを 3D へ自動外挿してはならない。**

---

## 0. 役割分担（計算環境）

| 段階 | 場所 | 担当 |
|---|---|---|
| **2D 探索・測定器開発・粗い地図・薄い/局所 3D の事前計算** | サンドボックス（2D＋α） | Claude（設計）|
| **正式 3D Room の計算・昇格・登録** | リポジトリ（AI 用計算場所を含む） | Codex / Claude Code |

**サンドボックスの成果は「候補発見」であって「正式成功」ではない。** 正式 Room は t=0 から終了まで 3D。サンドボックスは 3D 昇格の**リスク評価と事前計算**までを担い、正式 3D はリポジトリ側で走らせる。

---

## 1. 各段階の役割

**2D（探索）：** 仮説候補の高速探索、数値安定性確認、測定器開発、粗いパラメータ地図、明確な失敗領域の除外、線形不安定性の確認。→ **2D 成功は候補発見。**

**局所 3D（事前）：** 渦線の面外不安定性、界面曲率、3D 再結合、小領域での流体不安定性、2D にない崩壊方向。

**低解像度全体 3D：** 大域循環、長距離相互作用、多数構造の競合、有限サイズ効果、全体境界条件。

**正式 3D：** 十分な格子収束、十分な界面解像度、複数 seed、保存則/収支、再現可能な manifest、出力 checksum。→ **正式な到達 Level 判定・Room 昇格・正式結果。**

**原則：リポジトリ上の正式な Room は、最初から最後まで 3D で実行された結果でなければならない。**

正式 3D と認めない：2D 結果を押し出して立体表示／2D 断面を回転／2D に奥行きノイズ／2D 成功形状を 3D 初期条件に置く／局所 3D だけで全体 3D 未実行。

## 2. 次元移行監査（Dimension Transfer Audit）

2D 候補を全体 3D へ上げる前に必ず通す。

### A. 次元依存性の登録
```yaml
dimension_risk:
  topology_codimension_change: high
  inverse_cascade_dependency: true
  out_of_plane_escape: high
  vortex_reconnection: high
  curvature_instability: medium
  expected_transferability: low
```

### B. 2D 固有機構の検出（該当すれば高リスク）
2D 特有の逆エネルギーカスケード ／ 面内へ閉じ込められた渦 ／ 3D では避けられる衝突 ／ 2D だけのトポロジカル保護 ／ **面積保存を体積保存として扱っている** ／ 断面だけで閉じる境界 ／ 3D で許される追加自由度の無視。

### C. 線形解析（可能なモデル）
2D と 3D の分散関係を比較：最も成長する波数、**面外モードの成長率**、不安定帯域、臨界パラメータ、長波長不安定性、短波長数値不安定性。**2D で安定でも 3D 面外モードが不安定なら、全体 3D 昇格前に棄却または再設計。**

### D. 無次元量を合わせる（係数をそのままコピーしない）
Reynolds 数・Péclet 数・Rayleigh 数・Damköhler 数・Cahn 数・Weber 数・Mach 数・拡散長・反応時間・**界面幅/格子幅**・系サイズ/相関長。

### E. 薄い 3D スラブ試験
少数の z 層を持つ 3D スラブに、2D 状態へ微小な面外摂動を加え：面外へ崩れるか／渦が切れるか／新しい不安定モードが成長するか／構造が維持されるか。**薄いスラブ成功を正式 3D 成功とは呼ばない。**

### F. 局所 3D パッチ試験
重要領域（欠陥交差・界面・高曲率・くびれ・渦線・高散逸）だけを高解像度で。

## 3. 3D 失敗も保存する

3D で失敗した候補を削除しない。
```yaml
status: rejected_in_3d
failure:
  category: out_of_plane_instability
  first_failure_time: 42.6
  detected_by: [z_mode_growth, vortex_reconnection]
```

**2D 成功・3D 失敗は、次元の違いが物理を変えた重要な発見**として系統樹へ残す（消さない）。

## 4. サンドボックス（2D＋α）でやるべきこと

Claude 側の事前計算の具体：
1. 始原条件の 2D 探索（Level をどこまで自然に育てる条件かを地図化）。
2. 複数 seed で 2D 再現性を確認。
3. **§2 の A–D（次元リスク登録・2D 固有機構検出・線形解析・無次元量）を事前評価**し、3D 昇格リスクを付けて渡す。
4. できる範囲で薄い 3D スラブ／局所 3D の事前計算。
5. **正式 3D 判定・全体 3D はリポジトリ側（Codex/Claude Code）へ、リスクレポートと共に渡す。**
