# AGENTS.md — Aeterna-Genesis で働く AI への共通規則

このリポジトリで作業する AI（Claude Code / Codex / その他）が**最初に読む**ファイル。詳細は `docs/` の各文書へ。

> **このリポジトリ（Genesis-Room-）固有の一節：** これは Genesis Room 3D 計算の確認用リポジトリ（本体
> Aeterna-Genesis とは別、`experiments/e001–e036` は入らない）。Codex は `genesis/_template.py` の型に従い、
> `common/` の diagnostics・io を使って、`results/` に §B 形式で結果を保存する。以下の規律は本体と共通。

---

## このプロジェクトは何か

**完成した生命・身体・脳・宇宙を人間が設計して組み立てるのではない。** 未分化な始原状態に**物理法則と局所的な関係だけ**を与え、時刻 0 から発展させたとき、差・構造・局在・運動・循環・境界・全体性・成長・分岐・継承が**どこまで結果として自然に現れるか**を観測する研究環境。

**進歩の定義：** 機能を一つずつ積み上げることではない。**より深い創発まで自然に到達する始原条件を発見したこと**を進歩とする。

---

## 三層構造

1. **Evidence Library**（`experiments/e001–e036`）：現象・法則・測定器・成立条件・負の結果を蓄積した証拠庫。**削除・番号変更しない。** 生命の部品表ではなく、物理辞書。
2. **Genesis Rooms**（`rooms/`）：一つの始原条件から **t=0 から中断なく**発展させ、どこまで自然に育ったかを記録した再現可能な宇宙。パラレル宇宙として分岐保存。
3. **AI Genesis Lab**（`ai_lab/`）：完成形を作らず、**始原条件を探索**する自動実験者。候補を 2D→局所3D→全体3D と昇格。

---

## 最重要規律（3つ）

1. **毎回 0 から始める。** 全ての run は時刻 0 から。
2. **先へ進むために途中へ機能を追加しない。始原条件を変えて Level 0 から再実行する。** ⚠️ **Level 4 の後に Level 5 の機能を"足して"はならない。** より深くしたいなら、始原条件の何を変えれば最初からそこまで育つかを探す。
3. **「これは物理だ」と言える誠実さを、成果数・見栄えより優先する。** 詳細は `docs/PHYSICS_INTEGRITY.md`。

## 役割分担（計算環境）

| 場所 | 役割 | 担当 |
|---|---|---|
| **サンドボックス（2D＋α）** | 始原条件の 2D 探索、測定器開発、粗いパラメータ地図、薄い/局所 3D の事前計算、明確な失敗の除外 | Claude（設計）|
| **リポジトリ（正式 3D）** | 正式 3D Room の計算・昇格・登録。AI 用計算場所を含む | Codex / Claude Code |

**2D 成功は候補発見であって正式成功ではない。正式 Room は t=0 から終了まで 3D。** 2D→3D は自動外挿せず、次元移行監査（`docs/DIMENSION_POLICY.md`）を通す。

## AI が絶対にやってはいけないこと（要約）

- 求める完成形（身体・膜・器官・脳）を直接与える／初期条件に置く。
- 中心制御装置・全体を監視する外部 oracle・生死を決める条件分岐・分裂の外部命令。
- 結果を特定形状へ収束させる強制項、完成形との画像類似度を目標にする。
- **評価ゲート・初期条件・方程式に、結論と同型の因果を埋め込む（第8監査。`docs/PHYSICS_INTEGRITY.md` §「第8監査」）。**
- 保存量を壊す補正、結果を見た後の成功条件変更、成功 run だけ残す、可視化データを物理結果として使う。
- 過去の Room・テンプレート・結果・失敗を削除する。
- 「生命」「脳」「身体」「宇宙」等の強い語を、正式定義を満たさずに使う（測定量から判定。限定名称「◯◯候補」を使う）。

## AI が変更してよいもの（始原側のみ）

初期分布・ノイズ強度/相関・物理定数・拡散/反応係数・外部流入量/分布・初期対称性・局所相互作用範囲・空間/時間スケール・保存則を満たす局所法則候補。**法則変更はパラメータ変更と明確に区別**（`mutation_type: law_variant`、より厳しい監査）。詳細は `docs/AI_EXPERIMENT_POLICY.md`。
